Clymb AI Provenance API

Complete API documentation. Base URL: https://api.clymb.ai

Authentication: All endpoints (except /health) require an API key via the X-API-Key header.

Authentication

curl -X POST https://api.clymb.ai/classify \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"text": "..."}'

POST /classify

Classify content as HUMAN, MACHINE, or INDETERMINATE using statistical analysis.

Request

FieldTypeRequiredDescription
textstringYesContent to classify (min 50 characters)

Response

{
  "classification": "MACHINE",
  "confidence": 0.72,
  "distance": 2.45,
  "features": {
    "compression_ratio": 0.68,
    "char_entropy": 4.21,
    "unique_word_ratio": 0.93,
    "avg_sentence_length": 24.5,
    "sentence_variance": 2.1,
    "trigram_uniqueness": 0.97
  }
}
FieldDescription
classificationHUMAN, MACHINE, or INDETERMINATE
confidence0.0 to 1.0 — how certain the classification is
distanceMahalanobis distance from human reference distribution
featuresStatistical features extracted from the text

POST /gate

Gate training data — admit or reject with provenance tracking. Use this in your training pipeline to block contaminated data.

Request

FieldTypeRequiredDescription
textstringYesTraining data content
generation_depthintegerNoEstimated AI generation depth (default: 1)
min_fidelityfloatNoMinimum acceptable quality score 0-1 (default: 0.5)
sourcestringNoData source identifier
block_machinebooleanNoBlock high-confidence machine content (default: false)

Response

{
  "admitted": true,
  "reason": "passed_all_gates",
  "fidelity": 0.9975,
  "generation_depth": 1,
  "n_collapse": 246,
  "origin": {
    "classification": "HUMAN",
    "confidence": 0.65
  },
  "data_hash": "a1b2c3d4e5f6g7h8",
  "timestamp": "2026-04-13T20:00:00"
}

POST /decay

Predict quality decay using the McCaffer-Bovill activation potential. Returns the full decay curve and collapse predictions.

Request

FieldTypeRequiredDescription
generation_depthintegerYesGeneration depth to query
theta_0floatNoDecay rate parameter (default: 0.1)

Response

{
  "query_depth": 5,
  "quality_at_depth": 0.9752,
  "n_collapse_50pct": 246,
  "n_collapse_10pct": 741,
  "theta_0": 0.1,
  "model": "McCaffer-Bovill F(n) = cos²(θ₀√n / 2)",
  "patent": "GB2607042.5 Section 4.2",
  "decay_curve": [
    {"n": 0, "quality": 1.0},
    {"n": 1, "quality": 0.9975},
    ...
  ]
}
The McCaffer-Bovill Formula: F(n) = cos²(θ₀√n / 2) where n is the generation depth and θ₀ is the contamination rate. This is not a curve fit — it's derived from the physics of activation potentials (McCaffer PhD, Glasgow, 1977). Patent GB2607042.5 Section 4.2.

POST /certify

Issue a cryptographically signed provenance certificate. HMAC-SHA256 sealed, tamper-evident, legally defensible.

Request

FieldTypeRequiredDescription
textstringYesContent to certify

Response

{
  "content_hash": "a1b2c3d4e5f6...",
  "origin": "MACHINE",
  "confidence": 0.85,
  "provenance_depth": 1,
  "predicted_fidelity": 0.9975,
  "watermark_verified": true,
  "issued_by": "Clymb Ltd",
  "patent": "GB2607042.5",
  "signature": "f8e9d0c1b2a3...",
  "timestamp": "2026-04-13T20:00:00"
}
Certificate signatures are computed using HMAC-SHA256 with your account's signing key. To verify a certificate independently, recompute the HMAC over the certificate fields (excluding the signature itself) using your key.
